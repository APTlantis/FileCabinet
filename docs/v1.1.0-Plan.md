# Preview Pipeline V1

## Summary
Build a reliable local preview pipeline focused on deterministic thumbnails and cached preview assets, without adding PDF rendering or Windows shell-handler dependency yet. Generate preview assets during ingest/rescan and verify them during repair so the vault’s existing `thumbnails` folder becomes meaningful.

## Key Changes
- Add thumbnail metadata to artifacts:
  - `ThumbnailRelativePath As String`
  - `ThumbnailStatus As String`, using values like `Generated`, `Fallback card`, `Not applicable`, `Generation failed`.
- Add a `ThumbnailService` responsible for:
  - Generating scaled image thumbnails for image files.
  - Generating simple cached card images or metadata-backed fallback records for non-renderable formats.
  - Resolving absolute thumbnail paths from vault-relative paths.
  - Rebuilding missing thumbnails when requested by repair/rescan flows.
- Integrate generation into:
  - Ingest: after stored file creation, hash, and text extraction.
  - Rescan adoption: when cataloging existing vault files.
  - Repair: report missing thumbnail files and regenerate them when safe.
- Update preview behavior:
  - Prefer real thumbnail assets for images in the Preview tab.
  - Keep existing format-aware cards for non-renderable formats.
  - Show thumbnail status in Details so users can tell whether a preview is generated, fallback, or failed.
- Update docs to describe the preview pipeline accurately, including that PDF rendering and OCR remain deferred.

## Test Plan
- Unit test image thumbnail generation:
  - Creates a thumbnail under `thumbnails\yyyy\MM\`.
  - Stores a vault-relative thumbnail path.
  - Produces bounded dimensions suitable for preview UI.
- Unit test non-image artifacts:
  - Installer/archive/ISO artifacts receive a fallback thumbnail status without attempting image decoding.
- Unit test repair behavior:
  - Missing thumbnail files are detected.
  - Regeneration restores valid thumbnail metadata where possible.
- Regression test ingest/rescan:
  - Existing hash, text extraction, category, tags, and relation behavior remain unchanged.
- Run:
  - `dotnet test FileCabinet.Tests\FileCabinet.Tests.vbproj`

## Assumptions
- V1 optimizes for reliable local behavior, not broad file rendering.
- PDF first-page rendering, OCR, and Windows shell thumbnails remain deferred.
- Thumbnail generation happens during ingest/rescan, with repair able to regenerate missing assets.
- Existing catalog files remain backward-compatible by defaulting missing thumbnail fields to empty path and `Not applicable`.
